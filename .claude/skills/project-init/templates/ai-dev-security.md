## 3. 権限区分（3層）

| 層 | 内容 | 例 |
|---|---|---|
| AIが自律でやれる | 統制内の生成・調査・テスト実行 | 実装・設計書ドラフト・テスト実施（別エージェントが検証: Q-11） |
| 人間の承認が要る | 統制の変更・外部への影響 | 設定/MCP変更（PRレビュー）・工程ゲート通過・成果物の確認 |
| 人間にしかできない | AIに委ねない最終判断 | 本番適用・リスク受容・顧客提出・検収判定・契約/金額 |

## 4. データフローと漏洩リスク（何が漏れうるか・何で防ぐか）

| リスク | 前提（隠さず書く） | 防止策 |
|---|---|---|
| プロンプト経由の情報送信 | コード・docsの内容はLLM APIに送信される | 顧客データ・実キーを扱わせない設計（マスキング済みデータのみ）＋deny＋データ保持設定（{{プラン別}}） |
| 成果物への混入 | AI生成物に秘密情報・実名・内部パスが混入しうる | 第三者レビュー100%（Q-01）＋audit_pack.py 混入検査＋/delivery-package 最終検査 |
| 外部SaaS連携 | {{pm-sync接続先}} に課題情報が同期される | 接続してよい情報の範囲を確定（/pm-sync）・MCP許可リスト制 |
