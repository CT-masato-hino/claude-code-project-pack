# 階層レポート テンプレート（docs/04-reports/）

<!-- claude-code-project-pack の report-templates。全体→フェーズ→エージェントの3層で報告を正本化する。
     チャット上のサマリは揮発する — 同じ内容を必ずここへ残す。
     共通ルール:
       - 数値サマリーを必ず持つ（数値なしの合否宣言は無効 — qcd-standards 運用原則）
       - 人物表記はロールID/役割名のみ。実名を書かない
       - 上層は要約と数値だけ持ち、詳細は下層へリンクする（単層の長文にしない） -->

## 1. 全体報告書（docs/04-reports/overall/YYYYMMDD-overall.md）

```markdown
# 全体報告書 {{期間 or 節目名}}          版数: / 作成: {{ロールID}}

## レビュアー向けサマリ
- 前版からの差分: / 人間が判断すべきポイント（3〜5点）: / 影響するQ-ID・基準ID: / 最小読解セット:

## 数値サマリー
| 指標 | 値 |
|---|---|
| 完了工程 / 完了成果物数（D-01基準） | |
| テスト実施/合格（全レベル合算） | |
| 未解決 Must / Critical | |
| 主要QCD基準の状態（Q/C/D 緑黄赤） | |

## 実施概要（5行以内）
## フェーズ報告書へのリンク
- [{{工程名}}](../phase/YYYYMMDD-{{工程}}.md) — 判定: {{通過/条件付き}}
## 積み残し・次のアクション（issues/open-questions のIDで参照）
```

## 2. フェーズ報告書（docs/04-reports/phase/YYYYMMDD-<工程>.md）

leader のゲート判定とセットで作成必須（project-phase.md ゲート判定記録の「根拠」列からこのファイルを参照する）。

```markdown
# フェーズ報告書 {{工程名}}              版数: / 作成: {{ロールID}}

## レビュアー向けサマリ
（共通規約に同じ）

## 目的と対象範囲
## 実施内容（何をしたか。機能ID単位）
## 成果物（成果物台帳の該当行へのリンク・人間確認状態）
## 数値サマリー
| 指標 | 計画値 | 実測値 | 差分 |
|---|---|---|---|
| （成績書の数値サマリー表から転記） | | | |
## ゲート判定（project-phase.md の記録と一致させる）
- 判定: {{通過/条件付き通過/差し戻し}} / 判定者: leader / 根拠: {{QCD基準ID＋実測値}}
## 積み残し（条件・期限つき）
## エージェント報告書へのリンク
```

## 3. エージェント報告書（docs/04-reports/agent/YYYYMMDD-<agent>.md）

サブエージェントは「結論と要約のみ」をチャットへ返すのと**同時に、同じ内容を自分の報告書へ追記する**（メインのコンテキストは汚さず、記録は正本に残る — 汚染防止ルールと両立。Q-11監査の証跡になる）。

```markdown
# エージェント報告書 {{agent名}} {{YYYY-MM-DD}}

## 担当範囲（委譲されたタスク・機能ID）
## 適用したチェックリスト（エージェント定義のどの節を適用したか）
## 出力（作成/更新ファイルのパスと要旨 各1行）
## 判断と根拠（数値・出典つき。仮定を使った場合は Q-ID）
## 引き継ぎ（次の担当への申し送り・未完了事項）
```
